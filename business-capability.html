Business Capability

PM
Name
Album Catalog Management
Description
Maintain an authoritative catalog of music albums, allowing staff to add new releases, correct metadata, retrieve album details, and remove discontinued or erroneous entries — all against the existing /albums API.
2

User Story

Story
As a catalog administrator,
I want to create, view, update, and delete album records,
So that the music catalog remains accurate, complete, and free of stale or incorrect entries.
3

Acceptance Criteria

10 criteria
  • C
    A new album is saved and returned with a system-generated UUID id when all required fields are provided.
  • C
    A create request missing title returns HTTP 400 and does not persist a record.
  • R
    GET /albums returns all albums currently in the catalog as a JSON array.
  • R
    GET /albums/{id} with a valid existing id returns the exact album record for that id.
  • R
    GET /albums/{id} with a non-existent id returns HTTP 404.
  • U
    A POST /albums request with a valid id and changed artist returns the updated record reflecting the new value.
  • U
    A POST /albums request with an id that does not exist returns HTTP 404.
  • D
    DELETE /albums/{id} for an existing album returns HTTP 204 and the record is no longer returned by GET /albums.
  • D
    DELETE /albums/{id} for a non-existent id returns HTTP 404.
  • R
    GET /albums returns an empty array [] when the catalog contains no records.

CRUD Details

Dev
Create
Purpose

Add a new album record to the catalog.

Inputs
title * artist * releaseYear genre trackCount albumId
Validation

title non-null & non-blank · trackCount ≥ 0 if provided · releaseYear 4-digit numeric string if provided

Expected Behavior

Persists album; generates UUID id; returns full saved object.

Error Conditions

Missing title → 400 · Malformed JSON → 400

Read
Purpose

Retrieve one album by id, or the full catalog.

Inputs
id (path, single) — (list)
Validation

id must be a non-empty string.

Expected Behavior

Single: matching Album JSON. List: array of all albums, [] if none.

Error Conditions

id not found → 404 · Blank or malformed id → 404

Update
Purpose

Correct or enrich metadata on an existing album.

Inputs
id * title artist releaseYear genre trackCount albumId
Validation

id must exist · title non-blank · trackCount ≥ 0 if provided

Expected Behavior

Overwrites stored record with supplied values; returns updated Album object.

Error Conditions

id not found → 404 · Blank title → 400 · Missing id in body → 400

Delete
Purpose

Remove a discontinued or erroneous album from the catalog.

Inputs
id (path) *
Validation

id must be a non-empty string matching an existing record.

Expected Behavior

Removes record permanently; returns 204 no body; subsequent GET returns 404.

Error Conditions

id not found → 404

Edge Cases

7 cases
  • 1
    Create with trackCount: 0 — Zero is a valid integer; record must be persisted with trackCount: 0, not rejected.
  • 2
    Update changes only one field — Body contains id and only genre changed; all other fields must be preserved exactly, not nulled out.
  • 3
    Create with releaseYear in the future — e.g. "2099"; the API has no range constraint on year, so it must be accepted and stored as-is.
  • 4
    Delete then GET — DELETE /albums/{id} succeeds (204), then GET /albums/{id} must return 404, not a stale cached response.
  • 5
    Duplicate create — PUT /albums sent twice with identical field values (no id in body); two distinct records with different generated UUIDs must be created.
  • 6
    Update with unknown ID — POST /albums with a well-formed UUID never created; must return 404, not silently insert a new record.
  • 7
    GET /albums after all records deleted — Must return [] with HTTP 200, not 404.

Test Scenarios

8 cases
TC-01 PUT Create valid album
Input

PUT /albums with { "title": "Rumours", "artist": "Fleetwood Mac", "releaseYear": "1977", "genre": "Rock", "trackCount": 11 }

Expected Output

200 OK · response body contains all input fields plus a non-null id (UUID format)

TC-02 PUT Create — missing title
Input

PUT /albums with { "artist": "Fleetwood Mac", "releaseYear": "1977", "genre": "Rock", "trackCount": 11 }

Expected Output

400 Bad Request · no record persisted · GET /albums count unchanged

TC-03 GET Read existing album by ID
Input

GET /albums/{id} using the id returned from TC-01

Expected Output

200 OK · body matches the record created in TC-01 exactly

TC-04 GET Read non-existent album
Input

GET /albums/00000000-0000-0000-0000-000000000000

Expected Output

404 Not Found

TC-05 POST Update album artist
Input

POST /albums with full album body from TC-01, artist changed to "Fleetwood Mac (Reissue)"

Expected Output

200 OK · artist is "Fleetwood Mac (Reissue)" · all other fields unchanged · GET /albums/{id} reflects same change

TC-06 POST Update with unknown ID
Input

POST /albums with { "id": "00000000-0000-0000-0000-000000000000", "title": "Ghost Album", "artist": "Nobody", "releaseYear": "2000", "genre": "Pop", "trackCount": 1 }

Expected Output

404 Not Found · catalog unchanged

TC-07 DELETE Delete existing album
Input

DELETE /albums/{id} using the id from TC-01

Expected Output

204 No Content · subsequent GET /albums/{id} returns 404 · album absent from GET /albums

TC-08 GET List reflects all mutations
Input

Sequence: create album A → create album B → delete album A → GET /albums

Expected Output

200 OK · array contains exactly album B · album A is absent
